webpack222.config.js 和 package.json 的配置
 在webpack文件夹下面先创建webpack.config.js文件,通过Commonjs module.exports 导出,里面的path 依赖包，必须先安装, 然后在webstorm 中的 Terminal 中进入到对应的文件夹下面执行 npm init 生成 package.json 文件，然后在执行npm install 其他依赖执行。 ，找本地的好处是，比如服务器上是3.6，你本地全局是4.1，而直接配置build 后就是本地创建一个3.6中查找） 这样在Terminal中执行npm run build 就可以，本地安装npm install webpack@3.6.0 --save-dev 匹配版本，执行后会在package.json 生成devDependencies 开发依赖 (具体可以参考下面图片)
webpack中 css 的相关配置loader
 添加对应文件夹下的css 文件，然后参考https://www.webpackjs.com/loaders/css-loader/ 安装 npm install --save-dev css-loader ，完成后配置webpack.config.js 文件中 module 。 在参考 https://www.webpackjs.com/loaders/style-loader/ ，安装npm install style-loader --save-dev 最后运行 npm run build
1、遇到错误 The 'mode' option has not been set, webpack will fallback to 'production' ，参考https://blog.csdn.net/qq_34979346/article/details/99840181 解决方案：webpack222.config.js 配置中添加 mode: 'development' // 设置mode
2 、遇到错误 UnhandledPromiseRejectionWarning: TypeError: this.getResolve is not a function 当前版本太低，卸载后安装最新，参考https://www.jianshu.com/p/119a825d8bba 安装参考https://www.webpackjs.com/guides/installation/ (具体可以参考下面图片)
webpack中 less 的文件处理
 添加css对应文件夹下的less 文件，然后参考https://www.webpackjs.com/loaders/less-loader/ 在webstorm 中的 Terminal安装 npm install --save-dev less-loader less ，完成后配置webpack.config.js 文件中 module 配置 ,在入口文件index.js中引入，最后执行npm run build (具体可以参考下面图片)
webpack中 图片文件处理
 添加img对应文件夹下文件，然后参考https://www.webpackjs.com/loaders/url-loader/ 在webstorm 中的 Terminal安装 npm install --save-dev url-loader ，完成后配置webpack.config.js 文件中 module 配置 ,在入口文件index.js中引入，最后执行npm run build 注意：如果图片的大小 小于webpack.config.js 中配置 limit 的大小时，可以直接加载运行，要是大于limit 配置的值时，就要在Terminal安装 npm install file-loader --save-dev ,配置webpack.config.js中 output 加 publicPath:'dist/' 最后执行npm run build (具体可以参考下面图片)

webpack中 创建Vue 使用方案
 在webstorm 中的 Terminal安装 npm install --save-dev vue-loader vue-template-compiler,需要在webpack.config.js 文件中配置 const VueLoaderPlugin = require("vue-loader/lib/plugin"); 与 plugins ，然后重新 执行npm run build，
如果需要省略import后缀问题，列如import Cpn from "./Cpn"; 在webpack.config.js 中resolve 中添加 extensions:['.js','.css','.vue']
1、报错Module not found: Error: Can't resolve './vue/App' in 'G:\xmapp\htdocs\vue2\vuejs2\webpack\src' 解决方案1.可能是vue-loader 版本太高导致，修改一下package.json 文件中的"vue-loader": "^13.0.0" 版本号，然后npm install 执行npm run build 解决方案2、引入的时候写全 import App from './vue/App.vue'，不要省略最后的.vue (具体可以参考下面图片)

webpack-dev-server搭建本地服务器
 在webstorm 中的 Terminal安装 npm install --save-dev webpack-dev-server 参考：https://webpack.docschina.org/guides/development/#using-webpack-dev-server 需要在webpack.config.js 文件中配置 devServer,然后到package.json 中scripts 配置 "dev" : "webpack-dev-server",如果想运行后直接打开，就配置 webpack-dev-server --open
，然后重新 执行npm run dev ,此时在js中在添加修改就不需要重新运行,等测试好后最后执行npm run build 打包
(具体可以参考下面图片)
webpack-配置文件的分离
本地开发与发布的抽离，推荐 		目录下创建build文件夹，下面创建common公共文件，prod生成文件，dev开发文件 把之前webpack.config对应的配置放进去
在webstorm 中的 Terminal安装 npm install --save-dev webpack-merge 参考：https://webpack.docschina.org/guides/production/#setup
在prod 与 dev 文件中，需要引入 对应的const BaseConfig = require('./common.config');公共文件与 const { merge } = require('webpack-merge');
修改对应写法就行，具体参考图 ,删后webpack文件就行，修改配置package.json中指定 "build": "webpack --config build/prod.config.js", "dev": "webpack-dev-server --open --config build/dev.config.js"
因为配置文件路径变了，所以还要修改一下公共文件common中生成的路径 path ，最后重新 执行npm run build 与 npm run dev 就行
(具体可以参考下面图片)
